# services-bot-workers-backend

NestJS API serving bots, their workers, and the logs each worker produces. Built on Prisma + PostgreSQL.

## Stack

- **NestJS 11** — modular structure (`bots`, `workers`, `logs`, `prisma`)
- **Prisma 7** with the `@prisma/adapter-pg` driver adapter
- **PostgreSQL 16** (via `docker-compose.yml`)
- **Jest** for unit tests, **Supertest** for e2e

## Layout

```
src/
├── main.ts
├── app.module.ts
├── common/
│   ├── dto/pagination-query.dto.ts
│   └── interfaces/paginated-result.interface.ts
└── modules/
    ├── prisma/    # Global Prisma module
    ├── bots/      # controller / service / repository
    ├── workers/   # nested under /bots/:botId
    └── logs/      # nested under /bots/:botId and /bots/:botId/workers/:workerId
prisma/
├── schema.prisma            # Bot / Worker / Log models + indexes, BotStatus enum
├── seed.ts                  # loads ../../data/*.json into the DB inside a transaction
└── migrations/              # generated by `prisma migrate dev`
```

Each module follows the **controller → service → repository** split. Repositories own all `prisma.*` calls; services add cross-resource validation (e.g. `WorkersService.getWorkersForBot` calls `BotsService.getBotById` first to return 404 if the bot is missing).

## Setup

```bash
npm install
cp .env.example .env       # then edit DATABASE_URL / FRONTEND_URL if needed
docker compose up -d       # starts PostgreSQL on :5432
npm run prisma:generate    # generate the client
npm run prisma:migrate:dev # applies the init migration
npm run seed               # loads bots/workers/logs from ../../data/*.json
```

### Environment

| Variable       | Purpose                                                               |
| -------------- | --------------------------------------------------------------------- |
| `DATABASE_URL` | Postgres connection string (used by both runtime and Prisma CLI)      |
| `PORT`         | HTTP port, defaults to `3000`                                         |
| `FRONTEND_URL` | Allowed CORS origin. If unset, CORS reflects the request origin (dev) |

## Run

```bash
npm run start:dev      # nest start --watch
npm run start          # one-shot start
npm run start:prod     # node dist/main (after `npm run build`)
```

## Tests

```bash
npm run test           # unit tests (jest, *.spec.ts under src/)
npm run test:watch
npm run test:cov
npm run test:e2e       # e2e (separate config under test/)
```

Unit tests cover each module's repository and service plus `PaginationQueryDto`. Repositories are tested against a stubbed `PrismaService`; services mock their repository and any sibling services (e.g. `LogsService` tests mock `BotsService` and `WorkersService`).

## API

All routes are prefixed with `/api`. Pagination params are `page` (1-based) and `pageSize` (default 20, max 100). Skip is capped at `MAX_OFFSET = 10_000` to discourage deep paging.

| Method | Path                                      | Description                            |
| ------ | ----------------------------------------- | -------------------------------------- |
| GET    | `/api/bots`                               | Paginated list of bots                 |
| GET    | `/api/bots/:botId`                        | Single bot (404 if missing)            |
| GET    | `/api/bots/:botId/workers`                | Paginated workers belonging to the bot |
| GET    | `/api/bots/:botId/workers/:workerId`      | Single worker scoped to the parent bot |
| GET    | `/api/bots/:botId/logs`                   | All logs for the bot, with worker info |
| GET    | `/api/bots/:botId/workers/:workerId/logs` | Logs scoped to a specific worker       |

Successful list responses share the shape `{ items: T[], total: number }`.

## Prisma

```bash
npm run prisma:generate     # regenerate the client after schema changes
npm run prisma:migrate:dev  # create + apply a migration
npm run prisma:studio       # open Prisma Studio
```

The schema lives in `prisma/schema.prisma`. Indexes match the table queries: `Worker(botId, createdAt desc)`, `Log(botId, createdAt desc)`, `Log(botId, workerId, createdAt desc)`.
